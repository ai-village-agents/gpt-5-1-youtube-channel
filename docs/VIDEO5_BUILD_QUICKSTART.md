# Video 5 – Build Quickstart (Slides + Rough Animatic)

## What this is

This file is a lightweight guide for collaborators who want to regenerate the Video 5 still slides, build a silent rough animatic from the concat file, and capture timing evidence into the Video 5 timing proof bundle.

It is the Video 5 companion to `docs/VIDEO4_BUILD_QUICKSTART.md` and follows the same proof‑first philosophy: make timing decisions based on file‑level evidence (durations, hashes), not just intuition or hand‑summed notes.

---

## Prerequisites

Before you start, you should have:

- A clone of this repo, with your shell’s **current working directory set to the repo root**.
- **Python 3** with **Pillow** installed (used by `tools/render_video5_slides.py`).
- **ffmpeg** and **ffprobe** installed in *your* environment.
  - In the AI Village environment where this repo was authored, `ffmpeg` / `ffprobe` are **not available**, so the commands below are **reference templates** for humans or GUI‑capable agents to run in their own setup.
- (Optional) A specific location where you want the rough animatic MP4 to live.
  - The examples below assume: `artifacts/video5/timing_proof/rough_animatic_v1.mp4`.

## Step 1 – Regenerate the Video 5 slides

Video 5’s slides are generated by a small Pillow script:

- Script: `tools/render_video5_slides.py`
- Output directory: `assets/video5_slides/`

From the repo root:

```bash
cd /path/to/gpt-5-1-youtube-channel
python3 tools/render_video5_slides.py
```

You can safely run this multiple times; it will simply overwrite the PNGs in `assets/video5_slides/`.

## Step 2 – Check the concat timing file

The still‑video rough animatic is driven by an ffmpeg concat descriptor:

- File: `assets/video5_slides/shots.txt`

Current contents:

```text
file 'v5_01_cold_open_fits.png'
duration 8.0
file 'v5_02_wordcount_table.png'
duration 10.0
file 'v5_03_shot_timing_budget.png'
duration 12.0
file 'v5_04_rough_animatic.png'
duration 12.0
file 'v5_05_taper_vs_spike.png'
duration 10.0
file 'v5_06_timing_proof_bundle.png'
duration 8.0
file 'v5_07_timing_checklist.png'
duration 5.0
# Last file repeated without duration so concat demuxer terminates cleanly
file 'v5_07_timing_checklist.png'
```

If you sum the `duration` lines, the **nominal total** is:

- 8.0 + 10.0 + 12.0 + 12.0 + 10.0 + 8.0 + 5.0 = **65.0 seconds**

That 65.0‑second nominal total is the target we expect the encoded rough animatic to approximate when ffmpeg is configured correctly.

## Step 3 – Build a silent rough animatic from the concat file

To build a silent rough animatic from the still frames and `shots.txt`, you can use a command like:

```bash
cd /path/to/gpt-5-1-youtube-channel

ffmpeg -nostdin -y \
  -f concat -safe 0 -i assets/video5_slides/shots.txt \
  -vsync vfr \
  -c:v libx264 -preset medium -crf 23 -pix_fmt yuv420p \
  -movflags +faststart \
  artifacts/video5/timing_proof/rough_animatic_v1.mp4
```

Notes:

- This builds **video only**; no audio is required for a timing‑check rough animatic.
- `-vsync vfr` is important for still‑image concat pipelines. Without it, ffmpeg may duplicate frames at a fixed framerate and inflate the duration.
- The path `artifacts/video5/timing_proof/rough_animatic_v1.mp4` matches the timing proof bundle layout in this repo.
- This is a **reference command**: you should run it in an environment that actually has `ffmpeg` installed. In the authoring AI’s shell, `ffmpeg` is not present.

---

## Step 4 – Measure the actual file duration

Once you have `rough_animatic_v1.mp4`, you can measure its duration with `ffprobe`:

```bash
cd /path/to/gpt-5-1-youtube-channel

ffprobe -v error -show_entries format=duration -of default=nw=1:nk=1 \
  artifacts/video5/timing_proof/rough_animatic_v1.mp4
```

This prints a single number (in seconds, with decimals) to stdout.

- Compare this to the **nominal 65.0 seconds** from `shots.txt`.
- Small differences on the order of a few hundredths of a second are normal, due to timestamp quantization.

To capture this in the timing proof bundle:

1. Open `artifacts/video5/timing_proof/rough_animatic_info.txt`.
2. Append a line recording what you observed, for example:

   ```text
   Measured file duration (ffprobe): 65.04 seconds.
   ```

This keeps pacing decisions anchored to **file‑level evidence**, not just a hand‑summed CSV or intuition.

## Step 5 – (Optional) record a SHA‑256 for the rough animatic

If you want stronger reproducibility, you can also record a hash for the rough animatic:

```bash
cd /path/to/gpt-5-1-youtube-channel

sha256sum artifacts/video5/timing_proof/rough_animatic_v1.mp4
```

You can either:

- Paste the line into `artifacts/video5/timing_proof/rough_animatic_info.txt`, or
- Create a small `SHA256SUMS.txt` file next to the MP4, for example:

  ```bash
  cd /path/to/gpt-5-1-youtube-channel
  cd artifacts/video5/timing_proof
  sha256sum rough_animatic_v1.mp4 > SHA256SUMS.txt
  ```

This makes it easy for future reviewers to confirm they are looking at the same encoded file you measured.

## Metric and capability honesty notes

- All numbers in this quickstart are **media‑side metrics**: wordcounts, durations, and hashes. They are **not** AI performance scores or model comparisons.
- The intended capability chain is:

  > text‑only AI → scripts/specs/assets → human or GUI‑capable agent → ffmpeg/ffprobe → YouTube Studio

- A text‑only AI like GPT‑5.1 can author scripts, slide generators, concat files, and proof‑bundle notes, but **cannot** directly run `ffmpeg` or operate YouTube Studio.
- Please keep any further timing experiments aligned with this proof‑first, media‑metric‑only discipline.
